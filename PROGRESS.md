### 2026-07-04 — Playtest round 8 (highway onboarding) → utility-grace bypass fix

Played the new highway onboarding as a fresh player via the automation hooks: connected a road up to the highway gateway (founding tip auto-completed → firstZones, exactly as designed), then followed the tips (zone → wait for buildings). Result: 54 buildings / 141 pop grew, then the whole city mass-abandoned to population 0 within ~9s of game time — inside the 60s "utility grace" round 1 added to stop precisely this. (Two red herrings ruled out along the way: demand C = −1 was correct oversupply from my lopsided C-zoning, and the dead power network was my own line routed across a lake — power lines correctly can't bridge water; the honest ghost/Power overlay would guide a human.)

Root cause (read from `levelSystem`): the desirability `score` folds in `+10 if (powered && watered)`. Missing utilities drop that +10; with land value even mildly depressed (early industry/coal pollution — coal emits 30, land value 30→18 within ~6 cells), the raw score falls below `ABANDON_SCORE` (12), so abandonment fired on the FAST score path (`ABANDON_EVALS`=10 ≈8s) instead of the intended `UTILITY_ABANDON_EVALS`=75 (≈60s). The grace was effectively dead whenever land value < 24 — the exact onboarding case. Fix (TDD, one line in `src/sim/buildings.ts`): gate the score path on `!utilitiesBad` so a building missing power/water abandons ONLY on the long utility grace; the score path resumes once utilities connect (restored +10 lifts a merely-depressed score back over the line). Regression test `tests/sim/utilities.test.ts` "keeps the full utility grace where pollution depresses land value (onboarding)" (coal beside unpowered homes; assert 0 abandoned at 25 evals) — RED before (5 abandoned), GREEN after. 78 tests green; typecheck/lint/build green. Browser-verified both directions: the crash scenario now survives to ~75 evals (grace honored, then correctly abandons if never powered), and a prompt-utility play grows a healthy 24-building city (all powered+watered, 0 abandoned, commuters running) — screenshot captured via the canvas workaround. Docs: game-design.md abandonment rule rewritten (two independent paths; also corrected pre-existing drift — utility grace 30→75, recovery 5→3 evals); lessons.md entry added. Follow-up noted (not done): the power/water tips still only appear after buildings exist; surfacing them at zoning time would let players pre-wire, though the grace fix already makes the current flow survivable.
