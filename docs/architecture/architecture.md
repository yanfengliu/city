# Architecture

## Overview

Two threads. A Web Worker owns the simulation: a civ-engine `World` plus game systems, driven by the engine's own loop (20 TPS, speed-scaled). The main thread owns presentation: a Three.js scene, DOM HUD, and input. They speak a typed, JSON/structured-clone-safe protocol; the renderer never touches the World.

The worker does NOT use the engine's `ClientAdapter` or `RenderAdapter` classes. It drives the engine loop via `world.start()`/`setSpeed()`/`pause()`/`resume()`, submits commands directly via `world.submitWithResult` (custom `{type:'command', name, data}` envelope), and builds custom bulk/incremental messages from `world.onDiff`. Speed 0 maps to `world.pause()` тАФ never `setSpeed(0)`, which throws.

```text
main thread                      тФВ worker
                                 тФВ
input тЖТ tool logic тЖТ Command тФАтФАтФАтФАтФ╝тФАтФАтЦ╢ command envelope тЖТ submitWithResult тЖТ validators
                                 тФВ        World.step() @ 20 TPS ├Ч speed
Three.js scene тЧАтФАтФА RenderStore тЧАтФАтФ╝тФАтФАтФА onDiff тЖТ render messages (bulk + incremental)
DOM HUD       тЧАтФАтФА UiStore     тЧАтФАтФАтФ╝тФАтФАтФА frame payload (stats, demand, treasury, events)
overlay plane тЧАтФАтФА FieldStore  тЧАтФАтФАтФ╝тФАтФАтФА field chunks (on each field's cadence)
```

## Module boundaries

- `src/sim/` тАФ pure game logic on civ-engine. No DOM, no Three, no timers, no Math.random/Date.now. Everything deterministic and headless-testable. Exports `createCitySim(config)` (the world-factory) and shared types.
- `src/protocol/` тАФ message types both sides import. No logic beyond type guards. Every payload must be structured-clone-safe plain data.
- `src/worker/` тАФ thin host: instantiate world via factory, drive the engine loop (`world.start()`; 1x/2x/4x via `setSpeed`, pause via `pause()`/`resume()` тАФ never `setSpeed(0)`), submit commands via `world.submitWithResult`, and project `world.onDiff` into protocol messages over postMessage. No engine `ClientAdapter`/`RenderAdapter`; no game rules here.
- `src/rendering/` тАФ Three.js only. Consumes protocol messages into a RenderStore (entity views keyed by id+generation), builds/updates GPU resources. Never imports `sim/`.
- `src/ui/` тАФ DOM HUD (vanilla TS, no framework; keep it lean). Reads UiStore, dispatches commands via a single `submitCommand` funnel. Shared UI chrome lives in `hud-style.ts` so the top HUD, advisor, budget, inspect, legend, warning, toast, and milestone surfaces keep one light frosted-aqua civic planning skin without pulling render or sim code into UI.
- `src/app/` тАФ composition root: boot worker, wire stores, input тЖТ active tool тЖТ command, render loop.
- `src/harness/` тАФ opt-in dev playtest/replay adapters. `?record=1` enables the worker recorder and `window.__harness`; an ordinary localhost session keeps both off so diff cloning and the unbounded in-memory session sink do not tax normal play. The harness wraps the composition-root game surface, recorder messages, synthetic canvas/keyboard events, and HUD element clicks. Its CSS-sized JPEG is the WebGL map canvas only, without DOM HUD compositing. The civ-engine visual playtest host may read UI labels / map screenshot / text state and must use these UI-handler actions instead of the direct sim-command backdoor; separate full-browser checks cover trusted input, DOM hit-testing/occlusion, native pointer capture, and whole-page visuals.
- `src/persistence/` тАФ save/load: requests snapshot from worker, versions it, localStorage + file export/import; load path re-creates the worker world from snapshot.

## Worker protocol (v1)

Source of truth for implemented shapes: `src/protocol/messages.ts`. Notable divergences from the sketch below as built: `vehicles` and `pedestrians` are bounded full per-tick lists rather than upserts/removed; `field` messages carry `width`, `height`, and `defaultValue` alongside sparse cells; the inspect panel reads client-side building views (no `inspect` round-trip exists yet); `requestSnapshot`/`loadSnapshot` land in phase 6.

Client тЖТ worker: `{type:'command', name, data}` (one per player action; a custom envelope the worker hands to `world.submitWithResult` тАФ not the engine `ClientAdapter` тАФ with in-sim validation), `{type:'setSpeed', speed: 0|1|2|4}` (worker maps 1/2/4 to `world.setSpeed` and 0 to `world.pause()`/`resume()`; never `setSpeed(0)`, which throws), `{type:'requestSnapshot'}` (save), `{type:'loadSnapshot', snapshot, meta}`, `{type:'setFieldSubscriptions', fields[]}` (active overlay only), `{type:'inspect', entityId}` (info-panel round-trip).

Worker тЖТ client: `{type:'ready', gridW, gridH, terrain, seed}` (once; terrain includes normalized static `elevation: Float32Array`, `seaLevel`, water, and the initial tree mask; the renderer derives current trees locally as that mask minus road/building/structure footprints, so no tree message exists; the sim's landValue nearTrees input reads the same derived view sim-side), `{type:'zones', cells:[{i, zone}]}` (bulk; on zone/dezone and after load), `{type:'buildings', upserts: BuildingView[], removed: entityId[]}` (incremental, built from `world.onDiff` тАФ not an engine `RenderAdapter`), `{type:'vehicles', topologyVersion, list: VehicleView[]}` and `{type:'pedestrians', list:[{id,generation,fromCell,toCell,t,purpose,outbound}]}` (bounded full lists while active), `{type:'frame', tick, stats}` (every tick; includes active agents and completed shopping), `{type:'field', name, blockSize, cells}` (sparse block updates on that field's cadence, only when subscribed), `{type:'traffic', edges:[{id, bucket}]}` (on congestion-epoch change), `{type:'networks', power:{plants:[{kind,x,y,w,h,cells}], lineCells, plantCells}, water:{pipeCells, pumpCells}}` (on network recompute), `{type:'snapshot', snapshot, meta}` (save response), `{type:'commandRejected', name, code, message}` (surfaced as UI toast).

After `{type:'loadSnapshot'}` the worker responds with a fresh full sync тАФ `ready` (terrain regenerated from the restored seed), `roads`, `zones`, a full `buildings` upsert, `frame` тАФ the same sequence as initial boot.

Projected render views are minimal per archetype: growables use `BuildingView {x,y,w,h,zone,level,abandoned,powered,watered,...}`, services use `StructureView {x,y,w,h,service}`, and utilities use grouped/flat cell sets. Vehicles use `{id, edge, t, reverse}` plus the road topology version. Pedestrians project only the current exact-cell segment, purpose, and direction; their complete paths remain worker-local. Roads are not render-stream entities: the cell set and derived graph arrive on topology change.

## Rendering plan

- Terrain: one immutable `TerrainSurface` converts normalized cell elevation into a shared `(width+1)├Ч(height+1)` piecewise-linear heightfield. The same sampler drives terrain vertices and normals, coast skirts/sand, trees, roads/vehicles, zones, level foundations, utility geometry, ghosts, effects, textured overlays, terrain-aware picking, and `worldToScreen`; integer sim positions and `water[]` remain authoritative and elevation never changes simulation behavior. Water stays one flat recessed mesh, but `buildTerrainMesh` separately reads the raw below-sea elevation, normalizes `seaLevel - elevation` to a fixed 0.18 deep-water ceiling, averages incident water depths at shared corners, pins land-touching corners to shallow, and writes a continuous three-stop bathymetry color buffer into the existing single draw call. `WaterWaveMaterial` preserves that standard-lit vertex-color/shadow pipeline and injects two analytic wind-wave slopes into lighting normals only, then restores the flat receiver normal before Three.js computes shadow bias so cached tree/building shadow edges stay fixed; `CityScene` advances its wrapped renderer-local time uniform from the same presentation timestamp used by rAF and on-demand screenshots. No positions upload or shadow-map invalidation occurs per frame, and protocol, saves, sim, CPU terrain, and picking never see wave phase. The fixed highway footprint is graded to the build datum. Multi-cell walls extend down to the minimum footprint corner while roofs/details sit above its maximum, so upright stamps neither clip nor expose open foundations; inset road/zone/shore details split along the underlying cell-triangle seam instead of bridging its two planes. Saves keep the base seed and regenerate elevation, so save version 1 is unchanged. Trees deterministically partition the initial tree mask across conifer, broadleaf, and columnar low-poly archetypes with stable cell-hashed form and color.
- Roads: merged BufferGeometry rebuilt from the full `roads` message on topology changes; chunked dirty rebuilds are a later performance path once road payloads become large enough to justify them. Land asphalt, surface-detail, lane-marking, traffic, and vehicle layers sample the shared terrain surface. Bridge cells stay on the water-relative concrete causeway deck.
- Zones: a grouped renderer layer with translucent zone tint plus small low-opacity zone-colored planning parcels. Both meshes rebuild from the same `zones` payload and hide cells covered by growable, service, plant, or pump footprints, so empty planned lots read as civic planning overlays while occupied footprints stay visually clean; non-occupying lines and pipes do not occlude zoning.
- Buildings: five synchronized instanced layers per RCI zoneтАФwalls, roofs, rooftop details, multi-face windows, and semantic frontage assembliesтАФwith abandoned variants applied through instance color, plus service wall/roof/detail archetypes (fireStation, police, clinic, school). `building-archetype-geometry.ts` merges each zone's window planes and frontage boxes once, so literal doors, stoops, awnings, signs, loading bays, docks, and bollards add only one window and one frontage instance per building rather than per-feature scene objects; all 453 fixture buildings therefore remain a fixed 15 RCI batches. Windows scale with body height, while frontage height caps at the ground-floor presentation scale. Window batches are unshadowed and share the only emissive material; frontage and body batches remain non-emissive, and abandoned window matrices collapse so derelicts never glow while their doors/loading equipment stay visible and gray. Utility visuals are separate `NetworksView` geometry: grouped plant footprints share one level top/base, sparse power poles are terrain-anchored, each wire span rotates and stretches to join its two terrain-relative endpoints, and pumps/pipes sample the shared surface; underground pipe hints are hidden during normal play and enabled only with the Water overlay. Floating power/water warnings keep per-building state but render as at most three camera-facing `InstancedMesh` batches (power, water, both), so an outage across hundreds of buildings does not become hundreds of transparent draw calls. Their explicit cross-batch order is power, water, then combined, keeping overlap deterministic and prioritizing the most severe warning. Rebuild RCI and service instance buffers incrementally from `buildings` and `structures` messages.
- Vehicles: single `InstancedMesh` (capacity 600). Per frame: for each vehicle view, sample its edge polyline at `t` (+ per-tick lerp between last two sim states, renderer-owned), set instance matrix. Instance color by speed (whiteтЖТred) for readable congestion.
- Pedestrians: two fixed `InstancedMesh` batches (body/legs and head, capacity 256) share one dynamic instance-matrix buffer. Purpose colors distinguish commercial workers, industrial workers, and shoppers. Segment direction chooses the curb side, reverse travel naturally uses the opposite side, and the shared terrain sampler supplies height. Both batches disable casting and receiving shadows, so moving people never dirty the cached directional shadow map. Full-list reconciliation handles empty lists and generation reuse explicitly.
- Field overlays: one 64x64 (or 32x32 for blockSize 4) `DataTexture` per active field on a transparent `(width+1)├Ч(height+1)` grid draped above terrain; updated from sparse `field` messages. Only the active overlay is subscribed. The client-computed power/water overlay uses the same draped geometry and mirrors the sim's monotone conduction fixpointтАФstarting at source/conductor cells and repeatedly attaching in-range live growables plus servicesтАФso its connection-reach halo remains accurate during brownouts.
- Camera: `MapControls` (orbit/pan/zoom, angle-clamped); mouse and keyboard panning re-sample the focus height while shifting the camera equally, preserving view angle/distance across relief. Picking: clip the ray to the finite map/height bounds, traverse intersected cells front-to-back with bounded grid DDA, and test each cell's two visible triangles exactly; active off-map drags use a constant-work terrain-edge datum fallback before clamping. `worldToScreen` projects the same sampled surface so automated input stays invertible. Tools draw terrain-anchored ghost previews client-side and validate optimistically (authoritative validation in-sim; rejection тЖТ toast + ghost flash).
- Day/night (phase 6): renderer-local sun/ambient animation keyed to sim day fraction from `frame`. Directional shadows remain enabled, but the 2048┬▓ map is cached between caster changes and meaningful sun movement (1/1024 day, four sim ticks); tree/building/structure/bridge occupancy invalidates it immediately.

## Sim internals worth pinning

- System registry: single ordered registration list in `world-factory.ts` (deterministic registration order is a replay contract). Phases: input = command handlers only; update = growth/citizens/traffic/utilities; postUpdate = fields, demand, economy; output = stats projection.
- `createCitySim` takes explicit flags `{fieldsEnabled, utilitiesEnabled}` (both false until their phase тАФ fields in phase 4, utilities in phase 5). Disabled fields read landValue = 30, coverage 0, educated false; disabled utilities read powered = watered = true. The flags stay supported forever so early-phase scenario tests remain valid.
- Special footprint commands (services, power plants, and pumps) plan against execution-time occupancy: empty cells and growable R/C/I owners are allowed, while roads, water, and other special owners fail closed. The shared demolition path removes each intersecting growable's complete footprint with resident/job cleanup and rubble, preserves zoning plus non-occupying power-line/pipe overlays, and is reused by manual rectangle bulldozing. Replacement entities allocate before demolition to avoid immediate entity-id recycling; worker removals project component-specific diff removals so same-tick id reuse cannot erase a new view.
- Road graph, OccupancyGrid, Layers, and the PathRequestQueue are game-owned singletons living OUTSIDE World state тАФ with their authoritative source persisted where `world.serialize()` sees it: road cells live in a `roadCells` world-state record (small, written only on player action), pending trips as component data, and layer states as components on a dedicated singleton "mirror" entity (one component per layer, written only on that layer's recompute cadence). Layers go on the mirror entity rather than into `world.setState(...)` because world-state values are JSON-fingerprinted twice per tick by the engine, while component diffs are dirty-flag-only. OccupancyGrid and other derived maps are NOT mirrored anywhere. After `applySnapshot` (load/replay), `rebuildDerived(world)` reconstructs graph/occupancy/queue from entities and world state and layers from the mirror components. This function is the single choke point keeping derived caches honest.
- Traffic vs topology changes: edge identity is a geometry key (string of the two endpoint cells + the first path cell). On graph rebuild the sim remaps in-flight vehicle edge ids; vehicles referencing vanished edges despawn. Pedestrians use a separate exact-road-cell cache keyed by access-cell pair and topology version; a walker whose remaining path includes a removed road also despawns. Both cancellations restore the citizen's logical origin, increment `disconnectedTrips`, and cannot create retail sales. The congestion-aware vehicle PathCache keeps its existing monotonic `pathVersion` behavior.
- Determinism: strict mode on; the only RNG is `world.random()`; queues drain deterministically; congestion epochs and buckets are integer state in world state. CI (`.github/workflows/ci.yml`) runs the recorded-session determinism gate: `tests/sim/replay.test.ts` scripts a ~1,200-step build-out session and asserts `SessionReplayer.selfCheck().ok`.

## Performance budgets (from civ-engine benchmarks on this machine)

- Entities тЙд ~6,300 (2k citizens + тЙд600 vehicles + тЙд256 walkers + ~1тАУ2k buildings + services): engine ticks this within the 50 ms/tick budget at 20 TPS; current recorder and browser profiling must be refreshed whenever sim source changes.
- Pathfinding: road graph тЙд ~1k nodes; 8 path resolutions/tick budgeted through the queue; cached hits ~0.01 ms.
- Diff payloads at this scale ~25тАУ50 KB/tick worst case; fields throttled to their cadence and subscription.
- Render: instancing keeps draw calls тЙИ #archetypes + chunks + overlays тЙд ~50, including utility-warning storms. Target 60 fps. The renderer uses the stable `min(devicePixelRatio, 1.5)` backing-buffer ratio, which retains high-DPI detail while submitting 43.75% fewer pixels than DPR 2; rAF cadence never changes visual quality because it cannot reliably distinguish GPU pressure from display refresh, OS scheduling, or another busy browser. Shadows, water waves, overlays, tone mapping, and antialiasing remain enabled. Map-canvas harness JPEGs are downsampled to CSS dimensions so their image points, synthetic canvas input, picking, and canvas bounds stay in one coordinate space; they intentionally exclude DOM HUD/panel compositing, which full browser screenshots verify separately. `node scripts/benchmark-frame-pacing.mjs` takes the same OS-owned host lease as the historical GPU A/B so Codex threads cannot overlap measurements, builds production first, hashes the complete production source plus the consumed linked civ-engine package before/after the build and measurement, hashes the served bundle, and fails if either tree moves. It loads the SHA-pinned deterministic 936-person/453-building/88-vehicle fixture at 1280├Ч720, samples 600 live rAF frames for pause/1├Ч/4├Ч at device DPR 1 and 2, preserves raw per-frame pacing/quality/draw/long-task samples, and records main-thread duration, exact canvas quality, and simulation tick throughput. Acceptance requires the intended DPR 1 or capped DPR 1.5 buffer, mean тЙе58 fps, p95 тЙд18.5 ms, p99 тЙд25 ms, no three consecutive >20 ms frames, тЙе18 TPS at 1├Ч, тЙе72 TPS at 4├Ч, a stationary paused tick, and zero browser errors. The older isolated GPU-completion evidence (`benchmarks/results/2026-07-12-shadow-cache.json`) remains useful for the shadow-cache A/B: 0.1923 тЖТ 0.1123 ms mean, 53 тЖТ 34 calls, and 713,931 тЖТ 394,815 submitted triangles at paused DPR 1 on this RTX 4090 host.

## Testing strategy

- Contract tests per mechanic in `tests/sim/` using scenario helpers: build world тЖТ submit commands тЖТ step N тЖТ assert (growth happened, vehicle count > 0, field values in range, budget changedтАж).
- Pure-function unit tests for road-graph derivation, footprint search, L-path drag expansion, demand math.
- Determinism gate: synthetic playtest + replay selfCheck (see above) in `npm test`.
- Renderer/UI verified through the browser game-testing loop (`render_game_to_text()`, screenshots), not unit tests. `render_game_to_text()` reports: tick, population, treasury, demand, vehicle count, camera, active tool, and a coarse ASCII map тАФ enough for an agent to close the loop without pixels.
- Harness adapters have browser-free contract tests for marker conversion, replay inspection, and the civ-engine visual playtest host; they assert that the visual loop maps actions to `PlayerInput`/`advance` rather than `command`.

## Key decisions

| # | Decision | Why |
|---|---|---|
| 1 | Sim on civ-engine in a worker, renderer on protocol messages only | determinism, headless tests, main-thread frame budget |
| 2 | Grid-aligned roads; road graph derived, traffic on graph not cells | engine-native fit; graph keeps A* small and caches effective |
| 3 | Roads/terrain/fields as bulk messages, not per-cell entities | thousands of cells would drown the entity diff stream |
| 4 | Citizens = household entities (тЙИ3 pop each), capped vehicle agents | Skylines-credible agent traffic at browser-safe scale |
| 5 | Derived caches (graph/layers/occupancy) rebuilt from entities + mirror components via one `rebuildDerived` | save/load/replay correctness with non-serialized engine utilities |
| 6 | Vanilla-TS HUD, no React | small UI surface; avoids a framework dependency in the render path |
